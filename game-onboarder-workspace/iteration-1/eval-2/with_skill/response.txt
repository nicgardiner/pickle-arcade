All achievements sync to playerdata.json and display in launcher stats view.

4. GAMESDK INJECTION FOR MULTIPLAYER MATCH RECORDING
================================================================================

Injected call (at match end):

  GameSDK.recordMultiplayerMatch({
    gameId: 'speed_chess_v1',
    opponentPeerId: peer.id,
    result: 'win' | 'loss' | 'draw',
    duration: matchDurationMs,
    movesPlayed: moveCount,
    timestamp: Date.now()
  });

Placement: Inserted after final move validation, before match-end screen display.
Effect: Automatically updates pvp_wins, pvp_losses, matches_played, win_streak.
Sync: Bridge sends IPC message to main process; playerdata.json updated immediately.

Connection Hook Stubs:

  const peer = new Peer({ config: window.peerjsConfig });

  // On connection established:
  GameSDK.onMultiplayerConnect({ peerId: conn.peer });

  // On board state change (outgoing):
  GameSDK.syncBoardState({ position: boardFen, moveNumber: turn });

  // On incoming opponent move:
  GameSDK.recordOpponentMove({ from: sq, to: sq, notation: move });

  // On connection error:
  GameSDK.onMultiplayerError({ error: 'connection_lost' });

Message Handler: All PeerJS onData callbacks route through GameSDK bridge for
consistent stat tracking and reconnection logic.
